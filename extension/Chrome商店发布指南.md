## 第一步：注册开发者账号

### 1. 访问开发者控制台
打开浏览器，访问：
```
https://chrome.google.com/webstore/devconsole
```

### 2. 登录 Google 账号
使用您的 Google 账号登录（建议使用公司邮箱）。

### 3. 支付注册费
- 首次注册需要支付 **一次性 $5 USD** 开发者费用
- 支持信用卡、借记卡支付
- 此费用终身有效，可发布无限个扩展程序

## 第三步：打包插件

### 方式 A：手动打包（推荐）
1. 确保 `extension` 文件夹包含以下文件：
   ```
   extension/
   ├── manifest.json
   ├── popup.html
   ├── popup.js
   ├── content.js
   ├── background.js
   ├── icon.png
   └── 安装说明.md (可选，商店版不需要)
   ```

2. 将 `extension` 文件夹压缩成 **ZIP 文件**
   - macOS: 右键 → 压缩
   - Windows: 右键 → 发送到 → 压缩文件夹

3. 确保 ZIP 文件大小 < 100MB

### 方式 B：使用 Chrome 打包工具
1. 打开 `chrome://extensions/`
2. 开启"开发者模式"
3. 点击"打包扩展程序"
4. 选择 `extension` 文件夹
5. 生成 `.crx` 文件（但商店上传仍需 ZIP）

## 第五步：提交审核

### 1. 审核前检查清单
- ✅ 所有必填字段已填写
- ✅ 图标和截图已上传
- ✅ 隐私政策已设置
- ✅ 功能描述准确无误

### 2. 提交审核
- 点击 **"提交审核"**
- 审核时间：通常 **1-3 个工作日**
- 首次提交可能需要更长时间

### 3. 审核状态
- **待审核**: 等待 Google 审核
- **需要更多信息**: 按要求补充材料
- **已拒绝**: 查看拒绝原因并修改
- **已发布**: 🎉 成功上架！

## 第六步：发布后管理

### 1. 获取商店链接
发布成功后，您会得到一个唯一链接：
```
https://chrome.google.com/webstore/detail/[your-extension-id]
```

### 2. 分享给用户
用户只需：
1. 点击链接
2. 点击"添加至 Chrome"
3. 确认权限
4. 安装完成！

### 3. 更新插件
当需要更新时：
1. 修改 `manifest.json` 中的版本号（如 `1.0.0` → `1.1.0`）
2. 重新打包 ZIP
3. 在开发者控制台上传新版本
4. 提交审核
5. 用户会自动收到更新

## 📋 审核常见拒绝原因

### ❌ 权限过度申请
**问题**: 申请了不必要的权限  
**解决**: 只保留必需的权限（storage, activeTab, tabs）

### ❌ 隐私政策缺失
**问题**: 未提供隐私政策链接  
**解决**: 创建一个简单的隐私政策页面

### ❌ 功能描述不清
**问题**: 描述过于简单或夸大  
**解决**: 准确描述实际功能，不夸大宣传

### ❌ 图标不符合规范
**问题**: 图标尺寸错误或质量低  
**解决**: 使用标准尺寸的高清图标
